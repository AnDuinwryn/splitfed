What the Current Full Report Was Saying

The full HTML was an audit dashboard. It mixed formal fixed 0.5 results with diagnostic Youden thresholds, EENT-validation thresholds, SVD-independent static scaling, and summary-only rows.
The main experimental message is not "SplitAST-MAE wins everywhere". It is: static features lift EENT strongly, but create a severe domain-shift failure on SVD.
Audio-only and stable-static variants recover SVD to the 0.71-0.73 range, but they lose the high EENT accuracy that all 131D static features provide.
Rows using Youden or SVD-independent scaling are diagnostic evidence, not formal fixed-threshold claims, because the threshold/scaling protocol is no longer the same as fixed 0.5 cross-domain testing.

What Can Go Into the Paper

Main Result Table

  • Use Table 1 as the clean patient-level comparison.
  • Report fixed 0.5 as the formal fixed-threshold protocol.
  • State that SplitAST-MAE improves EENT with all131 features but fails under SVD domain shift.

Ablation Figure

  • Use Figure 1 or Figure 2 to show the EENT/SVD trade-off.
  • Audio-only and stable-static rows support the diagnosis that static features drive much of the cross-domain collapse.
  • Static-only 131D is the strongest evidence for corpus-specific static-feature bias.

Supplement Only

  • Use Youden and SVD-independent scaling as diagnostic/supplementary evidence.
  • Do not use them as the primary performance claim unless the paper explicitly defines that protocol.
  • The previous full HTML remains useful as an audit trail, not as a paper figure.
